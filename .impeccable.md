## Design Context

### Users

Engineers at the company using Terragon to run parallel coding agents in remote sandboxes. They're focused, multitasking across threads, and expect tools that stay out of their way.

### Brand Personality

Polished, smooth, clean.

### Aesthetic Direction

- **References**: Claude (claude.ai), Vercel dashboard, Linear
- **Theme**: Light/dark, existing OKLCH system
- **Anti-references**: Generic ChatGPT wrappers, Slack-like clutter
- The interface should feel like a premium developer tool — every pixel intentional, no visual noise.

### Design Principles

1. **Reliability is visible** — sync state between daemon and frontend should never feel uncertain
2. **Artifacts are first-class** — diffs, plans, and outputs deserve dedicated, clean presentation
3. **Smooth over flashy** — transitions and state changes should feel effortless, not attention-grabbing
4. **Dense but breathable** — engineers want information density, but with enough whitespace to parse quickly
5. **Zero jank** — no layout shifts, no stale states, no flickering between loading and loaded

## Findings considered and rejected

- **drizzle-orm SQL-injection advisory** — not exploitable here.
  `sql.raw(validatedTimezone)` (`threads.ts:1638-1639`, `usage-events.ts:97`) is
  safe: `validateTimezone` returns only IANA zones `Intl.DateTimeFormat` accepts,
  else `"UTC"` — no metacharacters reach the raw SQL. Verified independently.
- **better-auth 2FA-bypass critical** — the `twoFactor` plugin is not registered
  (`apps/www/src/lib/auth.ts:474-517`), so the advisory does not apply.
- **`@orpc/client` prototype-pollution / `handlebars` / `fast-xml-parser` CVEs** —
  CLI-only (locally run) or build-time-only (`sandbox-image`) with no untrusted
  input on a server request path. Covered by the CI audit-gate recommendation in
  #6 rather than individual upgrades.
- **Sandbox-provider duplication** — providers share a clean `ISandboxProvider`
  interface (`packages/sandbox/src/types.ts:79`); Daytona's larger size is
  provider-specific validation, not copy-paste. Not worth a consolidation plan.
- **Task-sharing / thread IDOR, proxy SSRF, daemon-event auth, GitHub/Linear
  webhook verification, AG-UI SSE replay ordering** — all checked and found
  correct (owner-scoped access, run-context-bound daemon tokens, timing-safe
  fail-closed webhooks, at-least-once replay dedupe). No finding.
- **delivery-loop v3 area** — does not exist on this branch (deleted in the
  runtime rewrite); AGENTS.md and prior memory referencing it are stale.

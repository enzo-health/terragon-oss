# PWA Audit & Improvement Plan — apps/www

Date: 2026-05-25
App: `apps/www` (Next.js **16.1.6**, App Router, React 19.1.2, Vercel)
Method: four parallel read-only auditors (manifest/installability, service worker/offline, next.config/perf, icons/assets/platform).

## TL;DR

The app is **installable today** (manifest + 192/512 icons + apple-touch + `appleWebApp` metadata all present) but is **not offline-capable** — there is no service worker anywhere in the repo. The manifest also has correctness bugs: maskable-only icons and a stale `theme_color` that disagrees with the live dark-mode meta tags. The single high-risk change is the service worker, because this app streams (SSE proxy routes + PartySocket WebSocket) and a careless cache strategy would break live chat.

## Findings by priority

### P0 — correctness bugs (safe, no new assets)

1. **Maskable-only icons.** Both manifest icons are `purpose: "maskable"` with no `"any"` variant → logo clips/zooms on every un-masked surface (install dialog, taskbar). Add `"any"` entries. `public/manifest.json:15,21`
2. **Stale, inconsistent `theme_color`.** Manifest says `#f8f5f0`; live metas say `#ffffff` (light) / `#2d2d2b` (dark) in `layout.tsx:114-123`; `theme-color-meta.tsx:13` uses yet another pair. Pick one light value, align the manifest to it. `public/manifest.json:7-8`
3. **Brittle favicon selector.** `hooks.tsx:93` uses `link[rel*='icon']` (substring) which also matches `rel="apple-touch-icon"`; works today only by DOM order. Tighten to `link[rel='icon']` or select by id.
4. **Missing `id` + `scope`.** No `id` → future `start_url` change orphans the install. Add `"id": "/"`, `"scope": "/"`.
5. **`orientation: "portrait-primary"`** on a landscape-first desktop tool. Drop it or set `"any"`.

### P1 — install quality (safe; some need new asset files)

6. **iOS status bar `black-translucent`** (`layout.tsx:22`) draws content under the clock without safe-area padding → unreadable status text. Use `"default"` unless insets are added.
7. **Notification badge wrong asset.** `use-notifications.ts:82-83` uses `favicon.png` (853×853, full color) as the `badge`; badges must be small monochrome. Needs a monochrome PNG.
8. **No `screenshots` / `shortcuts`** → minimal install dialog. Suggested shortcuts: New Task, Threads, Settings (verify routes). Needs screenshot PNGs + shortcut icons.
9. **`display_override` + `categories`** missing — minor polish.

### P2 — offline capability (HIGH RISK, architectural)

10. **No service worker.** Recommended: **serwist** (`@serwist/next`), the maintained successor to the abandoned `next-pwa`. Author SW as `src/app/sw.ts` so the caching matrix is explicit and typed.
    - **Hard rule:** `NetworkOnly` for all `/api/**`; `navigateFallbackDenylist: [/^\/api\//]`. SSE routes (`api/ag-ui/[threadId]`, `api/proxy/{anthropic,openai,google,openrouter}`, `api/run-setup-script/stream`) and auth must never be cached or fallback-rewritten.
    - **Precache** only hashed `_next/static/**`, icons, and an `/offline` page. NetworkFirst for navigations.
    - **`skipWaiting`/`clientsClaim` = `false`** — auto-activation would drop live WS/SSE mid-session. Prompt-to-reload instead.
    - **`disable` in development** — SW caching fights Turbopack HMR.
11. **Offline fallback page** `/offline` — static, no auth/data/realtime imports.
12. **Install prompt component** — client component listening for `beforeinstallprompt`, dismissible, suppressed when already standalone; iOS gets a one-time "Add to Home Screen" hint. Lives in `src/components/system/` near `ThemeColorMeta`.
13. **CSP / headers** (only when SW lands or CSP is introduced): `worker-src 'self' blob:`, `connect-src` must allow `wss:` (PartySocket) and `https:`, `img-src` must allow `data: blob:` + `cdn.terragonlabs.com`. Ship `Content-Security-Policy-Report-Only` first. Serve `/sw.js` with `Cache-Control: no-cache` and `Service-Worker-Allowed: /`; serve `/manifest.json` as `application/manifest+json`.

## Implemented (2026-05-25)

All three phases shipped on branch `feat/pwa-audit-improvements`.

**Phase A** — `public/manifest.json` rewritten: added `id`, `scope`, `display_override`, `categories`, `shortcuts` (Dashboard, Settings); `theme_color` → `#ffffff` (matches the light meta tag); dropped `orientation`; icons now declare both `any` and `maskable` purposes. Fixed the brittle `link[rel*='icon']` → `link[rel='icon']` selector in `src/components/chat/hooks.tsx`.

**Phase B** — `appleWebApp.statusBarStyle` `black-translucent` → `default` in `layout.tsx`. Added `src/components/system/install-prompt.tsx` (a sonner toast on `beforeinstallprompt`, dismissible, suppressed when standalone/already-dismissed), wired into `providers.tsx`. iOS A2HS hint intentionally skipped (avoids nagging Safari users; no install API there anyway).

**Phase C** — Hand-rolled `public/sw.js` (chosen over serwist because Next 16 builds with Turbopack, where serwist's webpack plugin can't emit the SW). GET-only; `/api/*`, `text/event-stream`, and range requests are never intercepted; navigations are network-first → cached `/offline`; static assets are stale-while-revalidate; no `skipWaiting`. Added `src/app/offline/page.tsx`, `src/components/system/service-worker-register.tsx` (production-only registration), and `headers()` in `next.config.ts` for `/sw.js` (no-cache + `Service-Worker-Allowed: /`) and `/manifest.json` content-type.

### Deliberately deferred

- **CSP** — the app has no CSP today, so the SW works without one. Adding a CSP blind risks breaking the realtime layer (`connect-src wss:`) and chat media (`img-src data: blob:`). Ship `Content-Security-Policy-Report-Only` first, then enforce. Not done here.
- **Asset-dependent items** — monochrome notification badge, `screenshots`, dedicated padded maskable art, iOS splash images. These need real PNGs; manifest `shortcuts` were added without icons (icons are optional).

- **Phase A (safe, now):** P0 fixes 1-5 + the favicon selector. Pure JSON/metadata, no new assets, no behavior risk.
- **Phase B (install quality):** fixes 6, 9, and the manifest `screenshots`/`shortcuts` structure (referencing assets to be added). Add install-prompt component.
- **Phase C (offline, gated):** serwist SW + `/offline` page + headers/CSP. Requires end-to-end verification that streaming still works token-by-token before merge.
